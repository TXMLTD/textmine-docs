When uploading document to Vault, the model will classify documents based on one 60 types and then extract the key terms based on the type. Previously, users would need to submit requests to TextMine to add new types.

It is now possible for users to create their own custom document types and associated tags in Vault.

## How to create a custom document type

Go to the organisation settings under the Vault section in the sidebar and create a new document type.

![Screenshot](/images/intercom/product-updates/10-april-2025/01-screenshot-2025-04-10-at-16-33-16.png)

You will then have to specify a name and category for the new document type as well as which Vault teams should have access to the custom document type.

![Screenshot](/images/intercom/product-updates/10-april-2025/02-screenshot-2025-04-10-at-16-33-48.png)

Avoid acronyms to improve the accuracy of the classification. For example, call a type Non-Disclosure Agreement instead of NDA. However, it can make sense to name a new type based on the acronym if the acronym or code is how the document type is referred to e.g. Form ADV, 10-Q, etc.

Once you've created your document type you will see the document type in the list. Click on the type to create tags for the new custom type.

![Screenshot](/images/intercom/product-updates/10-april-2025/03-screenshot-2025-04-10-at-16-33-59.png)

![Screenshot](/images/intercom/product-updates/10-april-2025/04-screenshot-2025-04-10-at-16-34-25.png)

You can enable/disable the tags for the type.

![Screenshot](/images/intercom/product-updates/10-april-2025/05-screenshot-2025-04-10-at-16-34-43.png)

## Predicting custom document types

Once a custom document type and associated tags has been enabled for a team, you can activate the custom document type pipeline from the team's Vault settings in the top right.

![Screenshot](/images/intercom/product-updates/10-april-2025/06-screenshot-2025-04-10-at-16-39-45.png)

Then enable custom document types.

![Screenshot](/images/intercom/product-updates/10-april-2025/07-screenshot-2025-04-10-at-16-35-07.png)

If this setting is enabled, Vault will then classify the uploaded documents amongst the custom types you have created.

![Screenshot](/images/intercom/product-updates/10-april-2025/08-screenshot-2025-04-10-at-16-35-37.png)

![Screenshot](/images/intercom/product-updates/10-april-2025/09-screenshot-2025-04-10-at-16-35-59.png)

You can also update the type of a document to a custom type by clicking on the type field of the document and selecting the corresponding type from the list.

![Screenshot](/images/intercom/product-updates/10-april-2025/10-screenshot-2025-04-10-at-16-36-39.png)

The list of types will correspond to your custom types only. To change to a TextMine default, please read the following section.

## Changing a default document type to a custom type and vis versa

To change a custom type to a default type, disable custom types in the team settings.

![Screenshot](/images/intercom/product-updates/10-april-2025/11-screenshot-2025-04-10-at-16-43-45.png)

When you select the type in the team view, you will then be able to see the list of default types.

![Screenshot](/images/intercom/product-updates/10-april-2025/12-screenshot-2025-04-10-at-16-43-53.png)

If you would like to change a default type to a custom type, select custom document types from the settings. When you click on the type the list will be limited to the custom types.

Whilst we are pro-active with fine-tuning the models, we might not have done training for new custom types and tags. Let us know if you are seeing inconsistent performance and we can improveme the models with additional fine-tuning.
