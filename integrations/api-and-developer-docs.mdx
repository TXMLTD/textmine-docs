## How to access the API

Use Public API V3 for new integrations and agent work. For self-service third-party agent provisioning in the multitenant non-enterprise deployment, start with [agent self-service signup](/integrations/third-party-agent-signup). Enterprise and dedicated deployments should contact TextMine support.

1. Confirm the user has access to the relevant TextMine organization, team, Vault, provider, workflow, record schema, or agent.
2. Create or request an API credential through agent self-service signup, TextMine support, or the developer access flow available for that organization.
3. Store the credential as an environment variable such as `TEXTMINE_API_KEY` or in the secret store for the agent client.
4. Send the credential as a bearer token on API and MCP requests.
5. Use the operation IDs, request schemas, and response schemas from the [API reference](https://developers.textmine.com/reference).
6. Poll asynchronous operations after uploads, provider imports, report generation, workflow runs, document reprocessing, task-agent work, and exports.

Use this shape for scripts and examples:

```bash
curl "<TEXTMINE_API_BASE_URL>/v3/..." \
  -H "Authorization: Bearer $TEXTMINE_API_KEY" \
  -H "Content-Type: application/json"
```

Use the base URL and environment shown in the developer hub, returned by discovery, or provided by the TextMine team. Do not paste bearer tokens into prompts, tickets, screenshots, source code, or shared documentation.

## MCP access

Use MCP when an agent client should discover and call TextMine tools from documentation or from an MCP-compatible tool surface.

| MCP surface | Use it for |
| --- | --- |
| TextMine MCP server | TextMine-hosted tools backed by Public API V3. Use this for product actions where TextMine exposes an MCP tool directly. |
| ReadMe MCP endpoint | Documentation-backed API discovery and execution from [developers.textmine.com/mcp](https://developers.textmine.com/mcp). |
| Workbench MCP providers | In-app provider scan/import flows that let Workbench browse external repositories and import files into a Workbench session. |

For setup, start with [MCP quickstart](https://developers.textmine.com/docs/textmine-mcp), then use the client-specific guide for Codex, Claude, Cursor/Windsurf, or ChatGPT/OpenAI. The multitenant hosted MCP URL is `https://app.textmine.com/mcp`; dedicated deployments should use their environment-specific MCP host.

Direct MCP clients should send:

```text
Authorization: Bearer tm_<key>
```

Platforms that own `Authorization` can use:

```text
X-TextMine-Api-Key: tm_<key>
```

## Agent self-service pattern

For autonomous onboarding:

1. Read [For agents](/getting-started/for-agents) for product routing.
2. Fetch `https://public-api.textmine.com/.well-known/textmine-agent.json` or `https://public-api.textmine.com/v3/third-party-agents/signup/discovery`.
3. Check `https://public-api.textmine.com/v3/third-party-agents/signup/availability`.
4. Generate a runtime RSA keypair and submit the public key with `POST https://public-api.textmine.com/v3/third-party-agents/signup/self-service`.
5. Decrypt `encryptedApiKey` inside the trusted runtime, store the `tm_...` key in a secret store, and keep it out of model-visible context.
6. Use Public API V3 or the TextMine MCP server with the stored key.

## How to choose the right layer

Use Mintlify when the question is:

- What is Vault, Legislate, Workbench, Workflows, Playbooks, Records, Agents, Scribe, or Integrations?
- What should users do in the app?
- How do product areas relate to each other?
- Which screenshots, release notes, and user workflows explain the current feature?

Use ReadMe when the question is:

- Which Public API V3 endpoint should I call?
- What is the exact request or response schema?
- Which operation ID should an agent use?
- How do I connect an MCP-compatible client?
- How do I safely execute provider import, report generation, workflow creation, metadata validation, records, or exports?

## API-backed agent pattern

For API-backed work, agents should follow this pattern:

1. Read [For agents](/getting-started/for-agents) for product routing.
2. Read the developer hub [For Agents](https://developers.textmine.com/docs/for-agents) page for API execution rules.
3. Use Public API V3 operation IDs from the API reference.
4. Discover IDs before mutating data.
5. Ask for confirmation before destructive, broad, credential, billing, or workflow-triggering actions.
6. Keep secrets out of logs, prompts, screenshots, commits, and docs.

## Current API focus

New integrations should use Public API V3. V2 remains available as legacy reference, but V3 is the agent-friendly surface for current public API work.
