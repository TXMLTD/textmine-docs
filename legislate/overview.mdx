Legislate remains a valid TextMine surface. In the current product model, it is the publication and team-document archive rather than another name for Vault.

## How Legislate relates to Vault

Vault is where documents are uploaded, processed, classified, tagged, reviewed, reprocessed, and analyzed. Once a document is validated, it can be published from Vault to Legislate.

Legislate is where published team documents can be listed and reviewed. Workbench treats Legislate/team-document inventory as a separate read target from Vault documents.

## When to use Legislate

- Review documents that have already been published from Vault.
- List published team documents for a business unit or team.
- Compare published Legislate documents with current Vault documents when the user explicitly asks to compare both.
- Use workflows that send documents to Legislate or respond when documents are published.

## When to use Vault instead

Use Vault when the task is about document processing, folders, document types, extracted tags, custom questions, Ask Vault, reports, reprocessing, or documents that have not yet been published.

## Workbench behavior

Workbench routes Legislate inventory questions to Legislate/team-document reading. It should not answer those questions by searching Vault unless the user explicitly asks for Vault, or asks to compare Vault and Legislate.
