---
title: "Workflows overview"
description: "Build reusable automations for document and data processes."
---

Workflows automate repeatable work in TextMine. A workflow has a trigger, ordered steps, optional conditions, and a run history.

![Workflow screenshot](/images/intercom/product-updates/26-may-2026/03-screenshot-2026-05-26-at-14-19-19.png)

## Triggers

Workflows can start manually, on a schedule, when documents are imported, processed, or published, when an email is received, or when someone submits an intake form.

## Actions

Workflow actions can run agents, upload to Vault, use Workbench-uploaded files, extract text, search EDGAR or fetch SEC filings, wait for document processing, apply playbooks, set or calculate values, reconcile data, compare against accepted outputs, enrich values, call APIs or webhooks, lookup or create records, collect analytics, sample populations, extract tables or line items, run AI reviews, fill templates, send reports, add reviewers or assignees, create alerts, request approvals, send to Legislate, import or submit to integrations, and send email.

## Conditions

Conditions include AI logic, decision policy matrices, entity checks, document type checks, and confidence checks.
